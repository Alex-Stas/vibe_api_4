# Weather Telegram Bot

## Описание проекта

Telegram-бот для получения и мониторинга погоды через OpenWeatherMap API.

Основные возможности:
- текущая погода по названию города;
- текущая погода по геолокации;
- прогноз на 5 дней по сохраненной геолокации;
- подписка на погодные уведомления;
- сравнение погоды в двух городах;
- расширенные метрики (включая качество воздуха и топ загрязнителей).

Проект рассчитан на диалоговый формат: бот ведет пользователя по шагам, запоминает состояние сценария и хранит пользовательские данные для подписок в JSON.

## Инструкция запуска

### 1) Требования
- Python 3.11+ (проект тестировался на Python 3.13);
- Telegram-бот, созданный через BotFather (`BOT_TOKEN`);
- API-ключ OpenWeatherMap (`API_KEY`).

### 2) Установка зависимостей

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 3) Настройка `.env`

Создайте файл `.env` в корне проекта:

```env
API_KEY=your_openweather_api_key
BOT_TOKEN=your_telegram_bot_token
```

### 4) Запуск бота

```bash
python bot.py
```

После запуска откройте диалог с ботом и отправьте `/start`.

## Общее описание команд и функций

### Команды в чате
- `/start` или `/help` — показать описание возможностей и главное меню.
- `//test_int` — сервисная команда для переключения интервала уведомлений:
  - обычный режим: проверка каждые 2 часа;
  - тестовый режим: проверка каждые 5 минут.
  Повторный вызов переключает режим обратно.

### Кнопки главного меню
- **Погода по городу** — запрос названия города, вывод текущей погоды.
- **Прогноз 5 дней (моё местоположение)** — показывает краткий список дней и детальный экран по выбранному дню.
- **Погода по геолокации** — запрашивает location и возвращает текущую погоду.
- **Погодные уведомления** — включает/выключает подписку кнопками «Да/Нет».
- **Сравнить города** — пошаговый ввод двух городов и таблица сравнения.
- **Расширенные данные** — подробный отчет (температура, давление, влажность, качество воздуха и др.).

### Подписки и хранение данных
- При включении уведомлений данные пользователя сохраняются в `User_Data.json`.
- Формат записи:
  `{"<user_id>": {"city": "...", "lat": ..., "lon": ..., "notifications": {"enabled": true, "interval_h": 2}}}`
- При отключении уведомлений по кнопке «Нет» запись пользователя удаляется из JSON.

## Расширенное описание работы модулей и функций в них

### `bot.py`

Главный модуль Telegram-бота:
- описывает интерфейс (клавиатуры и тексты);
- хранит runtime-состояния (`user_states`, `saved_locations`, `subscriptions`);
- обрабатывает входящие сообщения и callback-кнопки;
- запускает фонового worker-потока уведомлений;
- восстанавливает подписки из JSON при старте.

Ключевые функции:
- `main_menu()` — собирает главное меню.
- `location_keyboard(text)` — клавиатура с запросом геолокации.
- `notifications_keyboard()` — inline-кнопки «Да/Нет» для подписки.
- `format_timestamp(ts, timezone_shift)` — форматирование Unix-времени в локальный `HH:MM`.
- `get_weather_main_desc(weather)` — безопасно извлекает `main` и `description`.
- `format_current_weather(weather, title)` — формирует человекочитаемый блок текущей погоды.
- `build_daily_forecast(forecast)` — группирует 3-часовой прогноз по дням, считает min/max/avg.
- `forecast_keyboard(days)` — строит inline-список дней прогноза.
- `forecast_detail_keyboard()` — кнопки «назад/закрыть» для детализации.
- `notifications_worker()` — циклически проверяет активные подписки и отправляет уведомления:
  - смена погодного состояния;
  - изменение температуры на 3+ градуса;
  - сигнал о возможных осадках на следующий день;
  - в тестовом режиме (`//test_int`) отправляет тестовый пинг при каждой проверке.
- `_bootstrap_subscriptions_from_storage()` — поднимает активные подписки из `User_Data.json` при старте.
- `start_handler(message)` — обработчик `/start` и `/help`.
- `location_handler(message)` — сценарии, где требуется location.
- `send_extended_weather(chat_id, user_id, lat, lon, label)` — расширенный отчет по погоде и воздуху.
- `callback_handler(call)` — обработка inline-кнопок прогноза и уведомлений.
- `text_handler(message)` — маршрутизация пользовательского ввода по сценариям.

### `weather_app.py`

Модуль интеграции с OpenWeatherMap:
- выполняет HTTP-запросы к API;
- обрабатывает сетевые и HTTP-ошибки без падения бота;
- возвращает безопасные значения (`None`, `{}`, `[]`) при ошибках;
- хранит последнюю ошибку источника для ретраев и диагностики.

Ключевые функции:
- `get_coordinates(city, limit=1)` — геокодинг города в координаты.
- `get_current_weather(lat, lon)` — текущая погода.
- `get_forecast_5d3h(lat, lon)` — 5-дневный прогноз с шагом 3 часа.
- `get_air_pollution(lat, lon)` — данные компонентов загрязнения.
- `get_last_error(source)` — получение текста последней ошибки для указанного источника.
- `analyze_air_pollution(components, extended=False)` — классификация уровня загрязнения и топ-3 худших показателя.

Внутренние служебные функции:
- `_safe_request_json(url, params, source)` — безопасный HTTP-запрос с обработкой `429`, `4xx`, `5xx`, сетевых сбоев и плохого JSON.
- `_set_last_error(source, message)` — запись последней ошибки.
- `_safe_float(value)` — безопасное приведение к `float`.
- `_component_level(pollutant, value)` — вычисление уровня загрязнения по порогам.
- `_print_analyze_air_pollution(result)` — форматированный вывод анализа для локального запуска.

### `storage.py`

Модуль файлового хранения пользовательских данных в `User_Data.json`.

Функции:
- `load_user(user_id)` — загрузка данных одного пользователя.
- `save_user(user_id, data)` — сохранение/обновление данных пользователя.
- `delete_user(user_id)` — удаление записи пользователя.
- `load_all_users()` — загрузка всех записей (используется для восстановления подписок при старте).

Внутренние функции:
- `_read_all_users()` — безопасное чтение JSON с fallback на пустой словарь.
- `_write_all_users(data)` — безопасная запись JSON без проброса исключений наружу.
